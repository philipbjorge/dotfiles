# AGENTS.md

This repo is a macOS-first dotfiles repo managed with chezmoi.

## Architecture

- `bootstrap.sh` is the minimal first-run entrypoint.
  - It supports macOS only.
  - It installs Homebrew if needed.
  - It installs `git`, `chezmoi`, and `mise` with Brew.
  - If run outside this repo, it clones `https://github.com/philipbjorge/dotfiles.git` into `~/src/dotfiles`.
  - It runs `chezmoi init --apply "$repo_dir"`.
- Homebrew manages base macOS packages and apps.
- chezmoi manages dotfiles, templates, and post-apply scripts.
- mise manages language runtimes and developer tooling versions.
- zsh4humans owns the shell framework; chezmoi owns the generated startup files.

## Chezmoi Conventions

- Use chezmoi source-state names:
  - `dot_zshrc.tmpl` renders to `~/.zshrc`.
  - `dot_zshenv.tmpl` renders to `~/.zshenv`.
  - `dot_gitconfig.tmpl` renders to `~/.gitconfig`.
  - `dot_config/mise/config.toml.tmpl` renders to `~/.config/mise/config.toml`.
  - `Brewfile.tmpl` renders to `~/Brewfile`.
- Keep templates small and readable.
- Prefer machine-local data in `~/.config/chezmoi/chezmoi.toml` for profile differences:

```toml
[data]
profile = "personal"
name = "Philip Bjorge"
email = "github@philipbjorge.com"
install_gui_apps = true
```

## Shell

- Keep shell customization in `dot_zshrc.tmpl`.
- Keep `dot_zshenv.tmpl` close to the zsh4humans-generated bootstrap file.
- Do not run the zsh4humans installer from repo scripts. If refreshing upstream-generated files, run it manually, then import the generated files into the repo.
- `dot_p10k.zsh` is generated by `p10k configure`. Avoid hand-editing it unless the change is intentional and reviewed.

## Package Management

- Put macOS packages and casks in `Brewfile.tmpl`.
- Put language runtimes and versioned dev tools in `dot_config/mise/config.toml.tmpl`.
- Do not install `chezmoi` through mise for bootstrap. `chezmoi` and `mise` are base tools installed by Brew on macOS.
- Do not add Linux support casually. `bootstrap.sh` should continue to fail fast on non-macOS until a proper Linux path exists.

## Scripts

- `run_onchange_after_10-brew-bundle.sh.tmpl` should run Brew only on macOS.
- `run_onchange_after_20-mise-install.sh` runs `mise install`.
- Keep apply-time scripts idempotent and non-interactive.
- Do not put `gh auth login`, `p10k configure`, or other interactive flows inside `chezmoi apply`.
- `scripts/doctor` is the explicit post-bootstrap validation command.

## Validation

Run these after edits:

```sh
sh -n bootstrap.sh
sh -n scripts/doctor
sh -n run_onchange_after_20-mise-install.sh
zsh -n dot_zshenv.tmpl
zsh -n dot_zshrc.tmpl
zsh -n dot_p10k.zsh
```

`scripts/doctor` may fail before the machine is fully bootstrapped or before GitHub auth is refreshed. Treat its output as environment validation, not a pure repo syntax test.

## GitHub Auth

GitHub login is intentionally manual:

```sh
gh auth login
gh auth setup-git
```

The Git config enables the macOS Keychain credential helper. `gh auth setup-git` can be run after `gh auth login` to configure GitHub-specific credentials.
